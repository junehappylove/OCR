ASR能力

以下描述了如何使用灵云ASR能力。

1. ASR简介

ASR(Automatic Speech Recognition) 语音识别技术，其目标是将输入的语音信号转换为相应的文本或命令。

语音识别分为语法识别和自由说识别。

  1. 语法识别：开发者需要提供语法文件，系统在指定的语法范围进行识别。这种识别计算资源消耗少，识别率较高 但是要求用户的说话必须符合指定的语法。有限命令词汇的识别我们也归为语法识别， 因为可以将这些命令词的集合看做是一种特殊的语法。这种识别常用于命令操作、简单的意图控制等。

  2. 自由说：不限定用户说话的范围、方式和内容。自由说识别常常需要较大的语言模型作为支撑， 因此消耗计算资源较大。但这种识别可以用于诸如输入短信、输入微博、或者比较随意的对话系统等。 针对某些特定领域，也可以采用针对这一领域的语言模型作为支撑，因此对于说话集中在这个领域的内容会获得更好的识别率。 例如可以针对歌曲歌手的名称专门建立“音乐”领域的自由说模型。

2. 选择模块和能力

灵云可以提供云端和本地端的识别能力 无论使用云端或本地端的识别能力，基础模块 hci_asr都是需要的。当使用云端能力时，需要加上hci_asr_cloud_recog模块， 当需要使用本地端能力时，需要加上hci_asr_local_recog模块。

如果使用asr.local.grammar.chinese能力, 必须使用ispkgram_src_chinese.dat模型库文件。 ispkgram_src_chinese.dat模型库文件较小，但只提供了识别中文和英文字母的能力。例如 “打开QQ”，“播放”等。
如果使用asr.local.grammar能力，必须使用ispkgram_src.dat模型库文件。 ispkgram_src.dat库文件较大，但提供了识别英文单词的能力，可以识别诸如“打开Word”这样的短语或语法。

云端引擎目前包括了 asr.cloud.grammar 语法识别能力，以及asr.cloud.freetalk 自由说识别能力。 另外，针对自由说，还提供了两个常用的领域内自由说识别，一个是针对歌曲歌手名称的 asr.cloud.freetalk.music 能力， 一个是针对地名导航的 asr.cloud.freetalk.poi 能力。

注解
在Android平台下，提供了 armeabi 和 armeabi-v7a 两个版本的标准模块。如果能确认用户设备是armv7a以上CPU， 则可以使用armeabi-v7a中的hci_asr_local_recog.so，这样识别速度更快。

3. 语法识别

使用语法识别时，启动Session时，需要指定本地语法识别能力(asr.local.grammar, asr.local.grammar.chinese)或云端语法识别能力(asr.cloud.grammar)。

3.1 词表识别

最简单的就是给定一些命令词表，ASR在这些词汇集合中进行识别。

char * pszRecogConfig = "audioFormat=pcm16k16bit,encode=none,grammarType=wordlist";
// 必须是utf-8编码, 有些平台下可能需要转码
char * pszWordList = "播放\r\n暂停\r\n停止\r\n\打开\r\n下一首\r\n上一首\r\n";
ASR_RECOG_RESULT asrResult;
// pBuff是16K 16bit PCM 语音数据的缓冲区指针，nLen是其字节数
errCode = hci_asr_recog(nSessionId, pBuff, nLen, pszRecogConfig, pszWordList, &asrResult);

在配置串中指定grammarType=wordlist, 直接将词表传给hci_asr_recog()函数，灵云SDK即可在指定的词表范围内进行识别。

如果词表是一个文件，可以读入内存中再象上面一样传入识别函数，也可以直接在配置中指定isFile=yes， 则可以将文件名传入。

char * pszRecogConfig = "audioFormat=pcm16k16bit,encode=none,grammarType=wordlist,isFile=yes";
ASR_RECOG_RESULT asrResult;
errCode = hci_asr_recog(nSessionId, pBuff, nLen, pszRecogConfig, "wordlist.txt", &asrResult);

3.2 语法识别

词表识别是一种最简单的语法识别，还可以传入复杂一些的语法文件进行识别。

char * pszRecogConfig = "audioFormat=pcm16k16bit,encode=none,grammarType=jsgf";
char * pszGrammarData = LoadGrammarFromFile("grammar.txt");
ASR_RECOG_RESULT asrResult;
errCode = hci_asr_recog(nSessionId, pBuff, nLen, pszRecogConfig, pszGrammarData, &asrResult);

语法文件grammar.txt如下：

#JSGF V1.0;
grammar call;
public <call> = 打电话给 <contactname> ;
<contactname> = ( 张三 | 李四 | 王五 | 赵六 );

在配置串中指定grammarType=jsgf, 将语法文件内容读入后(必须是UTF8编码，以'\0'结束)传给hci_asr_recog()函数， 灵云SDK即按照指定的语法进行识别。也可以在内存中按照JSGF格式造出内存中的临时语法传入。

也可以配置isFile配置项，就可以直接将语法文件的文件名传入:

char * pszRecogConfig = "audioFormat=pcm16k16bit,encode=none,grammarType=jsgf,isFile=yes";
ASR_RECOG_RESULT asrResult;
errCode = hci_asr_recog(nSessionId, pBuff, nLen, pszRecogConfig, "grammar.txt", &asrResult);

本地语法识别接收的语法格式为JSGF(Java Speech Grammar Format，国际标准的语法格式)， 其格式的具体规范请参见：http://www.w3.org/TR/jsgf/。 除此之外还可以接收WORDLIST以及WFST格式的语法文件。 WFST格式是通过hci_asr_save_compiled_grammar()接口保存的语法文件。此格式的语法文件为经过编译后的语法文件， 可以通过WFST类型的方式载入，对于大的语法文件可以大大提高语法加载速度。

云端语法识别能力接收的是JSGF以及WORDLIST。 云端暂不支持WFST格式语法。

3.3 语法ID识别

除了指定语法内容外，还可以通过直接指定语法ID的方式进行识别：

char * pszRecogConfig = "audioFormat=pcm16k16bit,encode=none,grammarType=id,grammarId=10";
ASR_RECOG_RESULT asrResult;
// pBuff是16K 16bit PCM音频数据的缓冲区指针，nLen是其字节数
errCode = hci_asr_recog(nSessionId, pBuff, nLen, pszRecogConfig, NULL, &asrResult);

在配置串中指定grammarType=id, 并指定具体的grammarId, 灵云SDK即按照ID所指定的语法进行识别。 具体的grammarId有两种可能：

  1. 如果是云端语法识别，那么这个id是云端预置的ID。目前只有系统预置的一些ID，具体哪些可用可以咨询捷通华声公司。 将来可以通过开发者社区让开发者自行在后台添加并获得可以使用的ID。

  2. 如果是本地语法识别，需要事先通过 hci_asr_load_grammar() 加载语法，得到语法id，再进行使用。参见 3.4 本地语法加载 。

3.4 本地语法加载

在本地识别情况下，由于装载语法需要一定的编译时间，因此如果是事先能够确定的语法文件， 最好通过 hci_asr_load_grammar() 事先加载，然后一直通过其返回的id使用，加载语法数量上限为256。示例如下：

// 必须是utf-8编码, 有些平台下可能需要转码
char * pszWordList = "播放\r\n暂停\r\n停止\r\n打开\r\n下一首\r\n上一首\r\n";
// 装载语法
errCode = hci_asr_load_grammar("grammarType=wordlist", pszWordList, &nGrammarId);
// 识别
char szRecogConfig[1000];
sprintf(szRecogConfig, "audioFormat=pcm16k16bit,encode=none,grammarType=id,grammarId=%d", nGrammarId);
errCode = hci_asr_recog(nSessionId, pBuff, nLen, szRecogConfig, NULL, &asrResult);
// 可以使用此id进行多次识别
// 不再使用此id时，进行卸载
hci_asr_unload_grammar(nGrammarId);

针对jsgf语法文件的载入:

// 必须是utf-8编码，以'\0'结尾
char * pszGrammarData = LoadGrammarFromFile("grammar.txt");
// 装载语法
errCode = hci_asr_load_grammar("grammarType=jsgf", pszGrammarData, &nGrammarId);
// 识别
...
// 卸载语法
hci_asr_unload_grammar(nGrammarId);

也可以配置isFile配置项，就可以直接将语法文件的文件名传入:

// 装载语法
errCode = hci_asr_load_grammar("grammarType=jsgf,isFile=yes", "grammar.txt", &nGrammarId);
// 识别
...
// 卸载语法
hci_asr_unload_grammar(nGrammarId);

如果直接将语法传入 hci_asr_recog() ，实际上系统会每次都去进行编译，每次识别完成后再卸载语法， 因此会影响识别的速度。只有在每次的语法都是临时生成的情况下，例如每次是在某个歌手的歌名列表中选择， 事先不能确定，而且每次都有可能不一样，此时可以采用直接传语法，一步调用 hci_asr_recog() 的方式。

另外在使用 hci_asr_load_grammar() 加载语法后，可以使用 hci_asr_save_compiled_grammar() 对已加载的 语法文件进行保存。保存后的语法文件可以使用WFST类型的方式进行载入，对于大的语法文件可以大大提高语法 加载速度。

3.5 语法识别的选项

语法识别支持如下选项：

  1. audioFormat
    指定输入音频数据的格式。可支持的音频格式因ASR能力不同而不同，具体请参见 hci_asr_recog() 函数的说明。

  2. encode
    encode只对云端识别有效，表示音频传输时所用的压缩格式。目前云端语法识别只支持none(也即不压缩)， 或者alaw, ulaw压缩方式。由于speex和opus编解码库目前不对8K音频进行支持，因此云端语法识别目前也不支持 encode为speex和opus等音频压缩方式，将来统一到16K后会支持。

  3. candNum
    语法识别可以给出多个候选结果，这里定义了最大需要的候选结果数目。注意，有可能返回的候选结果少于这里定义的数目， 这是正常的。

grammarType, grammarId, isFile等选项也在上面进行了介绍，其余选项请参见 hci_asr_recog() 函数的说明。

4. 自由说识别

使用自由说识别时，需要在启动Session时，指定云端自由说识别能力 (asr.cloud.freetalk)或者某个领域的云端自由说识别能力。 使用本地自由说请与我们联系。

4.1 自由说识别

自由说的示例如下：

char * pszRecogConfig = "audioFormat=pcm16k16bit,encode=opus";
ASR_RECOG_RESULT asrResult;
errCode = hci_asr_recog(nSessionId, pBuff, nLen, pszRecogConfig, NULL, &asrResult);

可以看到，自由说识别和语法识别相比，只要Session启动时指定相应能力即可，在调用 hci_asr_recog() 时更为简单， 无需在配置项中指定 grammarType和grammarId，也无需传入grammarData参数。

4.2 自由说识别的选项

自由说别支持如下选项：

  1. audioFormat
    云端自由说识别能力目前支持输入pcm16k16bit, ulaw16k8bit, alaw16k8bit的数据。

  2. encode
    encode表示音频传输时所用的压缩格式。云端自由说识别支持none(也即不压缩)，alaw, ulaw, speex, opus等多种编码方式。 由于speex和opus压缩率比alaw,ulaw要大很多，应该在可能的情况下尽量使用这两种方式，以减少上传流量，提高响应速度。 具体配置参见 音频编解码库。

  3. candNum
    可以指定返回候选结果数目。但目前版本的自由说引擎只能返回一个候选结果，会忽略candNum 的配置项。

其余选项请参见 hci_asr_recog() 函数的说明。

5. 意图识别

使用意图识别时，需要在启动Session时，指定云端意图识别能力 (asr.cloud.dialog)。 目前不支持本地意图识别能力。

5.1 意图识别

意图识别的示例如下：

char * pszRecogConfig = "intention=weather,needContent=yes";
ASR_RECOG_RESULT asrResult;
errCode = hci_asr_recog(nSessionId, pBuff, nLen, pszRecogConfig, NULL, &asrResult);

可以看到，意图识别只要Session启动时指定相应能力即可，在调用 hci_asr_recog() 时 无需在配置项中指定 grammarType和grammarId，也无需传入grammarData参数。

5.2 意图识别的选项

意图识别支持如下选项：

  1. intention
    云端意图识别领域。云端意图识别能力目前支持的领域请咨询捷通华声或关注开发者社区。

  2. needContent
    指定是否需要意图识别的内容(yes/no)。

其余选项请参见 hci_asr_recog() 函数的说明。

6. 端点检测

灵云SDK中提供端点检测的能力，也即可以在传入的音频中检测到开始说话和停止说话的位置。 端点检测总是在本地端进行的，这样有两个用处：

  1. 检测到停止说话了，可以就触发识别过程。
  2. 可以只将有声音的这段数据，提交给云端，减少传输的流量。

6.1 端点检测过程

端点检测示例如下：

char * pszCheckConfig = "audioFormat=pcm16k16bit,vadhead=1600,vadtail=160";
ASR_VOICE_CHECK_STATUS eStatus = VAD_DETECTING;
// 检测标记，需要根据情况设置不同的值
// 第一次设置标识为 CHECK_FLAG_BEGIN, 中间的时候设置标识为 CHECK_FLAG_PROGRESS
// 最后如果主动终止，需要设置标识为 CHECK_FLAG_END
ASR_VOICE_CHECK_FALG eFlag = CHECK_FLAG_BEGIN;
while (true)
{
// 每次得到的数据放在 pBuf中，长度为 nLen
...
// 每次得到的数据放在 pBuf中，长度为 nLen
pBuf = ...
nLen = ...
isLastVoiceData = ...
...
if(eFlag == CHECK_FLAG_PROGRESS
&& isLastVoiceData)
{
//如果已经开启并且已经输入全部音频或输入最后一次音频，需要设置为CHECK_FLAG_END
eFlag = CHECK_FLAG_END；
}
ASR_VOICE_CHECK_RESULT check_result;
errCode = hci_asr_voice_check(nSessionId, eFlag, pBuf, nLen, pszCheckConfig, &check_result);
if(eFlag == CHECK_FLAG_BEGIN)
{
//此时需要调整状态为CHECK_FLAG_PROGRESS
eFlag = CHECK_FLAG_PROGRESS
}
if (check_result.eStatus == VAD_DETECTING || check_result.eStatus == VAD_VOICE_BEGIN)
{
// 正在检测中 或者 在当前的音频数据中检测到开始端点
}
if (check_result.eStatus == VAD_VOICE_END || check_result.eStatus == VAD_USER_STOP ||
check_result.eStatus == VAD_BUFF_FULL || check_result.eStatus == VAD_NO_VOICE_INPUT)
{
// 这几种情况，表示检测结束，跳出循环
break;
}
}
// 一旦检测结束，除非是VAD_NO_VOICE_INPUT, 否则 check_result 中的数据就是检测到的音频数据，
// 可以将此数据传给识别函数
if (check_result.eStatus != VAD_NO_VOICE_INPUT)
{
char * pVoiceBuf = (char *)check_result.pvVoiceData;
unsigned int nVoiceLen = check_result.uiVoiceDataLen;
errCode = hci_asr_recog(nSessionId, pVoiceBuf, nVoiceLen, pszRecogConfig, NULL, &asrResult);
}

检测结束的几种可能性如下：

  1. VAD_VOICE_END：正常检测到结束端点。
  2. VAD_USER_STOP：用户希望主动终止，设置eFlag为CHECK_FLAG_END。
  3. VAD_BUFF_FULL：如果总的输入的音频数据超过了maxSeconds配置项所配置的音频最大时长。
  4. VAD_NO_VOICE_INPUT：如果开头的静音时间太长，超过了vadHead配置项所规定的时间，表示没有检测到声音信号。

当前三种情况退出时，check_result中保存了检测到的音频数据，已经去掉了头尾的静音。

由于在噪音较大情况下，可能会一直检测不到结束端点，此时可以在UI上设计让用户主动终止的按钮，用户按下后， 设置eFlag为CHECK_FLAG_END，就可以停止端点检测过程。

6.2 端点检测的选项

端点检测支持如下主要选项：

  1. audioFormat
    目前只支持直接传入pcm格式的数据，也即只支持pcm8k16bit, pcm16k16bit两种输入。

  2. vadHead
    开始的静音检测，以毫秒为单位。当声音开始的静音超过此项指定的毫秒数时，认为没有检测到声音 (VAD_NO_VOICE_INPUT)。 如果此值为0，则表示不进行起点检测，无论检测到起点前的静音数据有多长，也不返回VAD_NO_VOICE_INPUT。

  3. vadTail
    末尾静音的长度，以毫秒为单位。检测到起点后语音数据出现静音并且静音时间超过此项指定的毫秒数时， 认为声音结束(VAD_VOICE_END)。如果此值为0，则表示不进行末端检测，用户必须使用 CHECK_FLAG_END 进行手工的停止或继续添加检测音频数据直到缓存区满。

7. 实时识别

本地语法识别与云端自由说识别支持实时识别模式。在实时识别模式下，调用 hci_asr_recog() 函数并不将传入的pvVoiceData当成所有的识别数据进行一次完整的识别，用户可以将音频数据分多次调用 hci_asr_recog()。 SDK内部会同时进行端点检测，检测到语音起点后会随时将数据传给引擎，因此可以在边录音的时候边进行引擎的识别工作， 在最后检测到末端时再返回最终的识别结果，这样能有更快的响应速度。

实时识别的示例如下：

char * pszSessionConfig = "capKey=asr.local.grammar,realtime=yes";
int nSessionId;
errCode = hci_asr_session_start(pszSessionConfig, &nSessionId);
char * pszRecogConfig = "audioFormat=pcm16k16bit,encode=none,grammarType=id,grammarId=10";
while (true)
{
// 每次得到的数据放在 pBuf中，长度为 nLen
...
ASR_RECOG_RESULT asrResult;
errCode = hci_asr_recog(nSessionId, pBuf, nLen, pszRecogConfig, NULL, &asrResult);
if (errCode == HCI_ERR_ASR_REALTIME_END)
{
// 正常检测到端点
//传入NULL和0，获取识别结果，退出循环
errCode = hci_asr_recog(nSessionId, NULL , 0 ,pszRecogConfig, NULL, &asrResult);
break;
}
else if (errCode == HCI_ERR_ASR_REALTIME_WAITING)
{
// 等待数据，继续循环
continue;
}
else
{
// 其它错误，一般不会出现，出现很可能是由于输入数据无效,或实时识别过程错误。
// 如果实时识别过程错误，则自动取消该次实时识别
// 否则可以手动终止本次实时识别, 然后退出循环
// 也可以继续循环，则下次hci_asr_recog()发送的数据会作为前次数据的延续
break;
}
}

在实时识别模式下，首先要在启动会话时配置 realtime=yes。然后调用 hci_asr_recog() 开启一次新的识别， 识别过程处于端点检测过程中返回HCI_ERR_ASR_REALTIME_WAITNG。如果音频传入过程中端点检测过程结束（没有音频输入，缓冲区满， 检测到末端）结束，则会返回 HCI_ERR_ASR_REALTIME_END, 需要用户继续调用该接口且传入参数中 pvVoiceData传入NULL或者uiVoiceDataLen为0，此时SDK将识别结果填写在psAsrRecogResult中，返回HCI_ERR_NONE。

如果在一次实时识别过程中，pvVoiceData传入NULL或者uiVoiceDataLen为0表示强制停止此次实时识别， 此时也会填充识别结果到psAsrRecogResult中并且返回HCI_ERR_NONE。

调用 hci_asr_recog() ，返回如下错误码则表示实时识别过程出错： HCI_ERR_DATA_SIZE_TOO_LARGE（20: 传入的数据量超过可处理的上限,单次识别音频数据经过端点检测处理后的大小超过320K）， HCI_ERR_ASR_ENGINE_FAILED（209: 本地引擎识别失败）， HCI_ERR_SERVICE_CONNECT_FAILED（8: 连接服务器失败，服务器无响应）， HCI_ERR_SERVICE_TIMEOUT（9: 服务器访问超时）， HCI_ERR_SERVICE_DATA_INVALID（10: 服务器返回的数据格式不正确）， HCI_ERR_SERVICE_RESPONSE_FAILED（11: 服务器返回操作失败），此时SDK将取消该次实时识别， 如果用户继续调用该接口传输数据则自动开启一次新的实时识别。 其他情况，如果返回非 HCI_ERR_ASR_REALTIME_WATING 并且非 HCI_ERR_ASR_REALTIME_END 的结果时， 该次调用发送的数据不会被使用，实时识别任务不取消，下次再调用 hci_asr_recog() 发送的数据作为 前一次发送的延续。 实时识别只使用每次新开启时的pszConfig和pszGrammarData数据，在实时识别过程中再传入的pszConfig和pszGrammarData 总是会被忽略。
