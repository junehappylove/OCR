配置串定义：
配置串是由"字段=值"的形式给出的一个字符串，多个字段之间以','隔开。字段名不分大小写。

字段 取值或示例 缺省值 含义 详细说明
imageType normal, screen normal 图像类型 normal: 普通图像
screen: 截屏图像
adjustImage yes, no no 是否直接做校正 yes: 校正
no: 不校正
binarize adaptive, global adaptive 二值化方法，对灰度或者彩色图像有效 global: 全局二值化算法，主要针对通过扫描、截屏等获取的图像
adaptive: 局部自适应二值化算法，主要针对通过拍照、摄像头等获取的受光照影响较大图像
localProcess yes, no yes 是否先做本地预处理，云端中文OCR支持该配置
现在主要是指二值化 		yes: 进行预处理
no: 不进行预处理


这里没有定义的配置项，会使用 session_start 时的定义。如果 session_start 时也没有定义，则使用缺省值
注解
配置项 adjustImage 决定检测到倾斜角后是否对图像进行倾斜校正，若adjustImage=no则只返回倾斜角度不改动图像数据；
若adjustImage=yes，则进行校正并根据倾斜角对图像数据进行校正。

HCI_ERR_CODE HCIAPI hci_ocr_init ( _MUST_ _IN_ const char *  pszConfig)

灵云OCR能力 初始化

参数
pszConfig全局配置串，ASCII编码，以'\0'结束
返回

HCI_ERR_NONE操作成功
HCI_ERR_SYS_NOT_INITHCI SYS 尚未初始化
HCI_ERR_OCR_ALREADY_INITOCR SDK多次初始化
HCI_ERR_CONFIG_INVALID配置参数有误，如设定值非法或格式错误等
HCI_ERR_CONFIG_DATAPATH_MISSING传入的不是OCR的capkey
HCI_ERR_CONFIG_CAPKEY_NOT_MATCH缺少必需的dataPath配置项
HCI_ERR_CAPKEY_NOT_FOUND没有找到指定的能力
HCI_ERR_LOCAL_LIB_MISSING本地能力引擎缺失必要的库资源
配置串定义：
配置串是由"字段=值"的形式给出的一个字符串，多个字段之间以','隔开。字段名不分大小写。

字段 取值或示例 缺省值 含义 详细说明
dataPath opt/myapp/ocr_data 空 光学字符识别本地资源所在路径 不使用本地能力时，此项不传
initCapKeys ocr.local 空 初始化所需的本地能力 多个能力以';'隔开，传入的能力必须合法，否则返回错误。忽略传入的云端能力key。如果不使用本地识别能力，此项不传。
fileFlag none, android_so none 获取本地文件名的特殊标记 不使用本地能力时，此项不传。详细说明参见下面的注释

Android特殊配置
当fileFlag为android_so时，加载本地资源文件(字典和编码库)时会将正常的库文件名更改为so文件名进行加载。 例如，当使用的库为file.dat时，则实际打开的文件名为libfile.dat.so，这样在Android系统下， 开发者可以按照此规则将本地资源改名后, 放在libs目录下打包入apk。在安装后，这些资源文件 就会放置在/data/data/包名/lib目录下。则可以直接将dataPath配置项指为此目录即可。

注解
若是云端识别，此配置串可以直接传NULL或者""

这里没有定义的配置项，会使用 session_start 时的定义。如果 session_start 时也没有定义，则使用缺省值。

注解
pRegions 如果为NULL或nRegionCount为0，则检查配置串中的 autoLayout, 如果是true，则后台进行自动版面分析， 如果是false，则将整幅图像当做一个完整的横排正文区域进行识别

pRegions 如果不为NULL且nRegionCount不为0，配置串不支持autoDeskew和autoLayout两个参数，因此总是不进行倾斜校正和版面分析, 而使用传入的pRegions作为识别区域
